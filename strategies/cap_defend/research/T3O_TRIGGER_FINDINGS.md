# 자산간 리밸 트리거 T3O(과대-트림) 연구 정리

작성 2026-06-16 KST. 상태: 조건부 채택 (라이브 선행 stress 통과 후). 매매코드 무변경.

## 1. 질문
현행 자산간(sleeve간) 송금 트리거 T1·T3U_can 에 T3O(한 sleeve 가 target 대비 과대 → 트림) 를
추가하면 BNB·SOL 아웃라이어 제외 유니버스에서 위험조정 성과가 개선되는가?

## 2. 셋업
- sleeve: 주식 60 / 코인현물 25 / 코인선물 15. 자금이동 수동(B안), 알림만.
- 현행 트리거: T1=half_turnover≥20pp(전체 reset) OR T3U_can=상대미달≥20% AND 카나리 ON(해당 sleeve 끌어올림).
- 실제 라이브 카나리 그대로 구현(프록시 아님):
  · stock = EEM SMA200 ±0.5% dead-zone (코드 확인. CLAUDE.md "SMA300/2%" 노트는 stale)
  · spot/fut = BTC SMA42 ±1.5% dead-zone
- BT: V25 라이브 파라미터로 BNB·SOL 제외 sleeve 곡선 3개 합성 → alloc overlay 일별 시뮬.
  2020-11~2026-05, 1371 거래일. 홀드아웃 금지 → 롤링 window rank-sum.
- 스크립트: research/bt_v25_t1_t3u_t3o.py (ablation), research/bt_v25_t3o_robust.py (robustness).

## 3. 핵심 발견 — T3O 카나리 방향
- on(과대+카나리ON) 과 none(무게이트) 결과 동일. 이유: 비대해진 sleeve 는 랠리해서 비대 →
  카나리가 거의 항상 ON. 따라서 게이트가 사실상 통과.
- off(과대+카나리OFF, T3U 대칭설계)는 +0.15 Cal 뿐 — 이미 폭락해 카나리 꺼진 뒤라 too late.
- 결론: T3O 는 T3U 의 대칭이 아니다. "이긴 sleeve 를 target 까지 차익실현 트림"(무게이트) 규칙으로 봐야 정합.

- 롤링 window rank-sum (W=252,378 × stride=15,21, n=244창): C1 이 Cal(avg-rank 2.05, top1 38.5%)·
  Sharpe(2.13, 35.2%) 양쪽 1위. C0 base 거의 최하위. 15/20/25% 임계 모두 평탄(단일점 의존 아님).
- 송금비용 stress 0/10/30/50bp: C1 3.05→3.03, base 대비 우위 유지. 비용 민감도 미미.

해석: 독립 alpha 라기보다 과대 성장한 고변동 sleeve 를 target 으로 되돌리는 변동성 수확/위험관리.
CAGR 0.5%p 희생하고 MDD·Sharpe·Cal 개선. 의사결정 기준은 alpha 여부가 아니라 수동 리밸 규칙으로
위험조정 성과가 안정적으로 좋아지는지 — 증거는 긍정적.

## 6. 라이브 선행조건 (채택 전 통과 필수)
1. 실행지연 stress: T+2 및 1/2/3/5거래일 지연, 50% 부분이행, 1회 skip, 월1회만 실행 가능 조건 재BT.
   통과기준: 지연/부분이행 후에도 base 대비 Cal·Sharpe·MDD 우위 유지, CAGR 훼손 허용범위 내.
2. 효과 분해: T3O 17이벤트 성과가 소수 에피소드/특정 sleeve/특정 regime 에 집중 안 됐는지. 트림 후 N일 수익률로 과열되돌림 실제 확인.
3. 대체 벤치 비교: 정기 월/분기 리밸, 단순 max-weight cap, vol-target overlay 대비 T3O 고유효과 분리.
4. BNB/SOL 포함/민감도: 제외로 인한 상방제한 과소평가 점검.
5. 비선형비용 worst-case: 세금·환전·김치프리미엄·출금한도·트래블룰 대기.
