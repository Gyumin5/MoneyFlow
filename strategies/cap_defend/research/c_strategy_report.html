Strategy C — Dip-Buy Mean Reversion

생성 2026-04-19 · V21 live 운영 상태 위 C 결합 검증 결과 스냅샷

1. 개요 — 전략이 하는 일

V21 롱 앙상블(주식 V17 / 코인 V21 D봉 3멤버 / 선물 V21 4h 3전략 L3)은 추세 추종에 특화되어 있어, 단기 급락 직후의 반등 구간을 잡지 못하고 cash로 들고 있는 경우가 많다. Strategy C는 V21이 비워둔 남은 현금에서만 동작하여 "급락 후 반등"을 짧게 먹는 보조 슬리브다.

설계 철학: V21은 "주인", C는 "남는 돈만 쓰는 보조자". V21이 다시 cash가 필요해지면 C는 자동 축소(shrink)되고, liquidation은 발생하지 않는다.

2. 시그널 로직 (c_engine_v5.py)

2-1. 진입 조건

1시간봉 기준, 최근 dip_bars 시간의 누적 수익률이 dip_thr 이하로 떨어지면 롱 진입.

cum_ret(1h × dip_bars) ≤ dip_thr   →   Long 진입 시그널 발행

2-2. 체결 가정 (보수)

  • 매수 가격: 시그널봉의 High (불리 체결)
  • 매도 가격: 청산봉의 Open (다음 봉 open, 지연 반영)
  • 거래비용: tx_cost=0.003 (편도 0.3%, 실전 Upbit 0.1~0.2% / Binance 0.04%보다 보수적)

2-3. 청산 조건

  • Take Profit: entry_px × (1 + tp) 터치 시 익절
  • Time Stop: tstop 시간 경과 시 강제 청산
  • Stop-Loss 없음: 반등 대기 (단 레버리지 wipeout 5% buffer는 작동)

3. V21과의 결합 구조 (핵심)

각 자산군 내 동일 계정에서 V21 우선, C는 남는 cash에서만 동작. m3_engine_final.py (현물) / m3_engine_futures.py (선물) 구현. 

alloc_base_eq       = V21_equity + C_delta             ← 슬리브 총자산
slot_margin         = cap_per_slot × alloc_base_eq     ← C 목표 margin (전체 대비)
remaining_margin    = V21_prev_cash × alloc_base_eq    ← V21 안 쓴 cash 한도
target_margin       = min(slot_margin, remaining_margin)
target_notional     = target_margin × leverage

if target_notional < cur_notional   →   shrink (부분 청산, TX 부과)
if target_notional > cur_notional   →   expand (더 매수)
중요: cap_per_slot은 전체 슬리브 자산 대비 비중이지 "남는 cash 대비"가 아니다. 예: cap=0.30, V21 cash_ratio=0.40 → C 목표 margin 30, 한도 40, 30 배정. V21 cash가 0.20으로 줄면 20으로 shrink.

유니버스 + 종목 관리

  • 유니버스: 당시 시총 Top 15 (stablecoin 제외, 역사적 universe json 기반)
  • 동시 보유: n_pick=1 1종목만
  • Swap: swap_edge_threshold=1 — 더 높은 cap_rank 신호 오면 기존 lot 교체

  1. 파라미터 grid sweep (240 configs): 현물/선물 각각 dip_bars × dip_thr × tp × tstop
  2. Train(~2023-12) / Holdout(2024+) 분리 재평가: 선택편향 없음 확인
  3. cap_per_slot 민감도: 0.03 ~ 0.333 13단계 (선물), 7단계 (현물)
  4. Top N 수익 이벤트 제거 ablation: 희소성 의존도 체크 (현물 불변, 선물 경미)

  6. AI 3자 리뷰 2라운드: Gemini + Codex 병렬 (cap 권고, stress 가이드)
  7. V21 baseline 검증: README Cal 2.89 CAGR 143% 일치 확인 (선물 L3 3멤버)
  8. 연도별 분해 + 초기 bootstrap: tail risk 예비 확인

7. 주요 발견

  1. 현물 C는 모든 구간(전구간/train/holdout)에서 단조 개선 → freeze 가능 수준
  2. 선물 C는 cap 딜레마 존재 — 낮은 cap(0.12): 전구간+Train 우위 / 높은 cap(0.30): Holdout 우위
  3. 반전: 선물 cap 0.10-0.12에서 Train MDD가 V21 단독보다 개선 → C가 drawdown 완충
  4. 희소성 의존 없음 (Top N 제거에도 Cal 유지)
  5. TX 스트레스 0.8%에서도 V21 단독 대비 우위 유지

8. 잔여 이슈 (자금 계층 설계 단계에서 다룰 것)

  1. 선물 cap 최종 선택 (0.12 / 0.25 / 0.30) — pool 설계에 종속
  2. 실전 주문 충돌 (V21 + C 동시 신호, 호가단위/최소수량 shrink 호환)
  3. High 진입 / Open 청산 가정 vs 실전 체결 괴리
  4. 반등 실패 dip 구간 평균/최악 보유시간 분포 확인
